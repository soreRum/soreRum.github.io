## Suricata Challenge: Detect Telegram API Connection Activity

**Goal**

Create a rule that will detect the packets related to the Telegram API communication.

**Rule**

```suricata
alert tls any any -> any any (
    msg:"Detected Telegram API connection Activity -- via Client Hello";
    flow:to_server;
    tls.sni;
    content:"api.telegram.org";
    nocase;
    sid:100001;
    rev:1;
)
```

**Thoughts**

It’s as simple as targeting **tls.sni**, the Server Name Indication.

Originally I included the **JA3 hash**, but realized it wasn’t helpful for detecting Telegram connection activity. JA3 fingerprints the client TLS handshake, not the destination (`api.telegram.org`).

JA3 focuses on the TLS client implementation, while **SNI directly identifies the domain being contacted**.

**Key takeaway**

- JA3 → destination independent  
- SNI → more direct and reliable indicator  

The challenge expected detection of **12 packets/alerts**.

## Suricata Challenge: Identify Speech Recognition Chrome Extension Payload in Network Traffic

**Goal**

Analyze the captured traffic and determine whether a Chrome extension payload is being transferred.

**Rule**

```suricata
alert http any any -> any any (
    msg:"Speech Recognition Chrome Extension Payload in Network Traffic --- Identified";
    flow:established,to_client;
    http.response_header;
    content:"application/x-chrome-extension";
    file.data;
    content:"Speech Recognition";
    content:"\"manifest_version\": 2";
    content:"\"name\": \"Speech Recognition\"";
    content:"Microsoft.CognitiveServices.Speech.core.dll";
    sid:100002;
    rev:1;
)
```

**Thoughts**

Following the HTTP/TCP stream revealed fragments of the extension manifest:

```
"name": "Speech Recognition"
```

The host:

```
msedge.b.tlu.dl.delivery.mp.microsoft.com
```

appears to be a **Microsoft Edge CDN endpoint used for extension downloads**.

Example request:

```
GET /filestreamingservice/files/2132f61f-f790-4ae6-a355-8cf9a1533800?P1=1757185982&P2=404&P3=2&P4=c1pT928%2fcNpn%2bhiPV%2feDJwgcmMghYDFStIXCZMBx0quHbu8wHa63x0JtAf%2fkqoVZhaG0phaVZr5%2fmgTnCtpSRw%3d%3d HTTP/1.1
```

The response:

```
Content-Type: application/x-chrome-extension
```

The transfer uses **byte range requests**:

```
Range: bytes=1120-2460
```

The response code:

```
206 Partial Content
```

suggests chunked delivery, which is consistent with **BITS download behavior**.

User agent confirms this:

```
User-Agent: Microsoft BITS/7.8
```

Embedded artifacts in the stream include:

```
manifest.json
Microsoft.CognitiveServices.Speech.core.dll
```

These markers indicate a **ZIP/CRX-style Chrome extension package**.

The challenge expected **1 alert**.

## Suricata Challenge: Detect SectopRAT C2 Beacon

**Goal**

Identify recurring outbound requests from an infected host that follow a consistent structure.

**Rule**

```suricata
alert http any any -> any any (
    msg:"SectopRAT C2 Beacon";
    flow:to_server,established;
    http.method;
    content:"GET";
    http.uri;
    pcre:"/\/wbinjget\?q=[A-Fa-f0-9]{32}(?:&|$)/U";
    classtype:trojan-activity;
    sid:100003;
    rev:1;
)
```

**Investigation Notes**

Observed domain:

```
vishneviyjazz.ru
```

PCAP IP:

```
185.95.159.55
```

The domain no longer resolves.

Another domain:

```
desk-app-now.com
```

Resolves to:

```
15.197.240.20
```

Nameservers:

```
NS1.VERIFICATION-HOLD.SUSPENDED-DOMAIN.COM
NS2.VERIFICATION-HOLD.SUSPENDED-DOMAIN.COM
```

Additional suspicious IP:

```
5.10.250.239
```

Traffic pattern observed:

```
/wmglb
```

followed later by:

```
/wbinjget?q=F3175C7E064B74698D8F67EDF4A4FEAA
```

with repeated:

```
HTTP/1.1 200 OK
```

This request repeats **~34 times**, indicating beaconing behavior.

Client:

```
10.8.15.176 → 5.10.250.239
```

The rule detects the **URI structure and parameter format**, specifically:

```
/wbinjget?q=<32 hex characters>
```

The challenge expected **34 alerts**.
